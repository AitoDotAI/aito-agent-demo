_predict
Each ticket is scored with aito._predict over historical tool-call resolutions. The model ranks the few relevant tools out of the whole catalog, with a calibrated confidence. High confidence fires the tool directly, no LLM call. Low confidence escalates to a human — and the routing desk is itself a prediction.
Example query
POST /api/v1/_predict { "from": "tickets", "where": { "text": "data slow when I travel abroad" }, "predict": "tool", "select": ["$p", "$why"], "limit": 4 } // → check_roaming (p ≈ 0.46) // below gate → escalate to a human
Verify yourself
Every routed call traces back to an Aito query. No separate model file, no retrain step — predictions come straight from the index, so a row added today is in the next prediction.
